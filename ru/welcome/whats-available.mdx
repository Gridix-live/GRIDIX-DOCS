Эта страница — короткий честный список того, что работает в GRIDIX прямо сейчас. Если статья по какой-то теме помечена «Скоро будет», это значит, что функция работает, но детальная инструкция по ней ещё дописывается. Вы можете пользоваться этой функцией — поддержка поможет с настройкой.

## Кабинет девелопера

- проекты любого типа: здание, объекты, генплан;
- лоты с импортом из Excel/CSV/Google Sheets;
- три статуса лота: доступен, забронирован, продан;
- цены в пяти валютах: RUB, USD, EUR, GEL, KZT;
- кастомные поля проекта и лотов;
- публичный каталог проекта на собственном поддомене или своём домене;
- виджет для встраивания на сайт, шесть режимов отображения;
- CRM с воронкой, задачами, тегами, источниками;
- автоматизация воронки: триггеры на вход в этап, по таймеру и по тегу;
- слияние дублей лидов;
- команда менеджеров с правами по проектам;
- аналитика просмотров, заявок и конверсии;
- email- и push-уведомления.

## Агентская сеть девелопера

- приглашение агентств, брокеров и агентов;
- модерация партнёров с четырьмя статусами;
- условия программы: ставка, срок фиксации, форс-мажор, эксклюзивность, территория;
- LeadLock: автоматическая фиксация клиента за партнёром;
- общий и персональный доступ к проектам;
- шаблоны договоров;
- финансовая аналитика по партнёрам.

## Кабинет агентства, брокера или агента

- доступ к проектам девелопера, к которым он открыл доступ;
- актуальные лоты, цены, статусы, материалы;
- ссылки для отправки клиенту;
- передача заявки в систему девелопера;
- финансовая аналитика по комиссиям.

## Партнёрская программа GRIDIX

- реферальная модель: ссылка с UTM, отслеживание кликов и регистраций;
- интеграторская модель: прямое ведение клиента;
- статистика клиентов, кликов, начислений;
- запрос выплаты вознаграждения;
- материалы для презентации платформы.

## Интеграции

- amoCRM — двусторонняя синхронизация заявок;
- Bitrix24 — синхронизация заявок, привязка сделок к лотам;
- email-уведомления о заявках;
- webhook (по запросу через поддержку).

## Безопасность и доступы

- роли в кабинете: владелец, менеджер с гибкими правами по проектам;
- блокировка пользователя без удаления истории;
- SSO (по запросу для командных тарифов).
