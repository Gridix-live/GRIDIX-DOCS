Пользовательские поля помогают хранить дополнительные параметры, которых нет в стандартной модели GRIDIX.

## Когда использовать

- вид из окна;
- корпус или секция;
- отделка;
- сторона света;
- специальные условия;
- внутренний ID из CRM или ERP.

## Типы полей

| Тип | Пример |
|---|---|
| Текст | "Секция A" |
| Число | "12" |
| Да/нет | "С отделкой" |
| Список | "Вид на море / город / двор" |

## Как настроить поле

<Steps>
  <Step title="Определите, где поле нужно">
    Решите, относится поле к проекту, лоту или внутренней работе команды.
  </Step>
  <Step title="Выберите тип поля">
    Для фиксированных вариантов лучше использовать список, а не свободный текст.
  </Step>
  <Step title="Добавьте поле до импорта">
    Если поле должно заполняться из таблицы, создайте его заранее и сопоставьте колонку при импорте.
  </Step>
  <Step title="Проверьте публичный вид">
    Убедитесь, что поле отображается там, где нужно пользователю, и не показывает внутренние данные.
  </Step>
</Steps>

## Рекомендации

<Warning>
  Не создавайте отдельное поле для данных, которые уже есть в стандартных полях лота. Это усложнит фильтры, импорт и аналитику.
</Warning>

<Tip>
  Перед массовым импортом согласуйте список пользовательских полей с командой продаж и CRM-интегратором.
</Tip>
