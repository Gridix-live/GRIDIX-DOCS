Генплан нужен, когда один проект состоит из нескольких корпусов, башен, очередей, домов, объектов или зон.

Вместо одного общего списка лотов проект получает структуру:

```text
Проект
├─ Подпроект 1
├─ Подпроект 2
└─ Подпроект 3
```

Каждый подпроект может иметь свои лоты, фасад, планировки, поля, цены и материалы.

## Когда использовать

- у проекта несколько корпусов;
- у корпусов разные этажи и планировки;
- нужно показывать генплан проекта клиенту;
- нужно встраивать отдельный корпус на сайт;
- нужно разделить данные между несколькими частями проекта.

## Что может делать пользователь

- активировать генплан;
- создать подпроект;
- импортировать лоты в подпроект;
- открыть редактор подпроекта;
- настроить визуальные зоны на генплане;
- показать генплан или отдельный подпроект в публичном каталоге.

## Для интегратора

Интегратор должен учитывать, что у проекта с генпланом может быть несколько публичных сценариев:

- весь проект с генпланом;
- список корпусов или объектов;
- список лотов;
- отдельный подпроект по `subProjectSlug`.

<Note>
  Информация готовится: нужно добавить пошаговую инструкцию с реальными скриншотами активации генплана, создания корпуса и настройки зон.
</Note>
