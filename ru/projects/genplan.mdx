Генплан нужен, когда один проект состоит из нескольких корпусов, башен, очередей, домов, объектов или зон.

Вместо одного общего списка лотов проект получает структуру:

```text
Проект
├─ Подпроект 1
├─ Подпроект 2
└─ Подпроект 3
```

Каждый подпроект может иметь свои лоты, фасад, планировки, поля, цены и материалы.

## Когда использовать

- у проекта несколько корпусов;
- у корпусов разные этажи и планировки;
- нужно показывать генплан проекта клиенту;
- нужно встраивать отдельный корпус на сайт;
- нужно разделить данные между несколькими частями проекта.

## Что может делать пользователь

- активировать генплан;
- создать подпроект;
- импортировать лоты в подпроект;
- открыть редактор подпроекта;
- настроить визуальные зоны на генплане;
- показать генплан или отдельный подпроект в публичном каталоге.

## Порядок настройки генплана

<Steps>
  <Step title="Создайте проект типа “Генплан”">
    На этапе создания выберите тип проекта, который поддерживает верхний уровень с корпусами или зонами.
  </Step>
  <Step title="Добавьте изображение генплана">
    Загрузите общий план проекта: территорию, корпуса, очереди, башни или зоны.
  </Step>
  <Step title="Создайте подпроекты">
    Для каждого корпуса, дома или зоны создайте отдельный подпроект.
  </Step>
  <Step title="Свяжите зоны с подпроектами">
    Отметьте область на генплане и укажите, какой подпроект должен открываться при клике.
  </Step>
  <Step title="Заполните подпроекты лотами">
    Импортируйте или добавьте лоты внутри каждого подпроекта.
  </Step>
  <Step title="Проверьте публичный сценарий">
    Откройте генплан как клиент: клик по зоне должен вести в правильный подпроект.
  </Step>
</Steps>

## Для интегратора

Интегратор должен учитывать, что у проекта с генпланом может быть несколько публичных сценариев:

- весь проект с генпланом;
- список корпусов или объектов;
- список лотов;
- отдельный подпроект по `subProjectSlug`.
