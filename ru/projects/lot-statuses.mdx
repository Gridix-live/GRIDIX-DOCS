Статус показывает, доступен ли лот для клиента и отдела продаж.

## Основные статусы

- `available` — доступен;
- `reserved` — забронирован;
- `sold` — продан.

## Где используются статусы

- в таблице лотов;
- в шахматке;
- в публичном каталоге проекта;
- в виджете сайта;
- в заявках и CRM-сценариях.

## При импорте

При импорте из таблицы GRIDIX помогает сопоставить значения из файла с системными статусами.

Например:

| В таблице | В GRIDIX |
| --- | --- |
| свободна | available |
| забронирована | reserved |
| продана | sold |

## Что проверить после импорта

- статусы в таблице сопоставились без ошибок;
- проданные лоты не показываются как доступные;
- забронированные лоты отображаются с правильным статусом;
- скрытые лоты не попадают в публичный каталог, если такой статус используется;
- тестовая заявка не отправляется по недоступному лоту, если это ограничено настройками.

<Note>
  Скриншоты нужно добавить: сопоставление статусов при импорте, таблица лотов со статусами, отображение разных статусов в публичном каталоге.
</Note>

<Note>
  Информация готовится: нужно добавить правила для нестандартных статусов клиента и примеры ошибок импорта.
</Note>
