Девелоперы передают через GRIDIX данные покупателей: имена, телефоны, email. Ниже — конкретные ответы на вопросы, которые чаще всего задают IT-службы и юридические отделы.

## Где хранятся данные

GRIDIX работает на инфраструктуре **Supabase** (облачная платформа на базе AWS). Данные хранятся в дата-центрах в соответствии с выбранным регионом. По умолчанию используется регион **EU West** (Ирландия).

Для Enterprise-клиентов доступно **частное облако** — изолированная среда с возможностью выбора региона, в том числе за пределами EU.

## Передача данных

Все данные передаются по зашифрованному соединению **HTTPS / TLS 1.2+**. Данные в состоянии покоя зашифрованы с помощью **AES-256**.

Данные вашего проекта изолированы от данных других клиентов на уровне базы данных — архитектура мультиарендности с row-level security.

## Кто имеет доступ к данным

- **Ваша команда** — только те, кому вы назначили роли в разделе **Настройки → Команда**.
- **Партнёры агентской сети** — видят только проекты и лоты, к которым вы открыли доступ. Персональные данные покупателей агентам не передаются.
- **Команда GRIDIX** — имеет доступ только для технической поддержки и устранения инцидентов. Данные не используются в коммерческих целях.

Полный список прав по ролям описан в статье [Команда и менеджеры](/ru/admin-panel/managers).

## GDPR и персональные данные

GRIDIX соответствует требованиям GDPR. Данные клиентов (заявки, контакты) хранятся в вашем аккаунте — вы выступаете контроллером данных, GRIDIX — процессором.

Это означает:
- вы отвечаете за получение согласия от клиентов на обработку их данных;
- GRIDIX обрабатывает данные только по вашему поручению;
- при необходимости вы можете запросить экспорт или удаление данных конкретного клиента.

<Note>
  Форма сбора заявок на виджете GRIDIX не содержит встроенного согласия на обработку персональных данных — вы добавляете его самостоятельно через кастомные поля или на уровне своего сайта. Если у вас вопросы по соответствию требованиям — напишите в поддержку.
</Note>

## Что происходит с данными при окончании подписки

Если подписка не продлена, проект переходит на паузу. Данные сохраняются в полном объёме — лоты, заявки, контакты, история. После оплаты всё восстанавливается без потерь.

Данные не удаляются автоматически при истечении подписки. Если вы хотите удалить аккаунт и все данные — напишите в поддержку: [@gridix_bot](https://t.me/gridix_bot).

## Экспорт данных

Вы можете экспортировать свои данные в любой момент:
- заявки и контакты — в формате Excel из раздела CRM;
- лоты — через интерфейс управления проектом.

Это важно: ваши данные не заблокированы внутри платформы.

## Логи и аудит

История изменений по заявкам фиксируется в карточке каждой заявки. Для Enterprise доступен расширенный аудит действий команды — обратитесь к персональному менеджеру.

## Уязвимости и инциденты

Если вы нашли уязвимость или подозреваете утечку — сразу напишите в поддержку: [@gridix_bot](https://t.me/gridix_bot). Команда GRIDIX реагирует приоритетно.
