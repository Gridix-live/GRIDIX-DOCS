# GRIDIX — внутренний план подготовки публичной базы знаний

Этот файл не должен попадать в публичную навигацию Mintlify. Он нужен команде GRIDIX как рабочий список: какие статьи дописать, какие скриншоты снять, какие видео записать и какие product/production-детали подтвердить перед публикацией.

## Принципы

- Публичная база знаний описывает только production-сценарии.
- `usertour` не переносится в базу знаний как отдельная документация. Это помощь внутри продукта.
- Если функция, тариф, выплата, срок или коммерческое условие не подтверждены, это не публикуется как факт.
- Агентская сеть девелопера, кабинет партнёра застройщика и партнёрская программа GRIDIX описываются как разные сценарии.

## Правила для будущих статей

Каждая публичная статья должна отвечать на шесть вопросов:

1. Что это за раздел?
2. Для кого он нужен?
3. Что здесь можно сделать?
4. Что нужно подготовить заранее?
5. Как выполнить действие по шагам?
6. Что делать дальше?

Компоненты Mintlify использовать только по задаче:

- `CardGroup` для выбора сценария и навигации.
- `Steps` для инструкций.
- `Tabs` для разных ролей.
- `Info`, `Note`, `Warning`, `Tip`, `Check` для важных пояснений.
- `AccordionGroup` для FAQ и ошибок.
- `Frame` для скриншотов, видео и схем.
- `Mermaid` для коротких процессов из 4-6 блоков.

## Единый шаблон страницы

Каждая статья в Центре помощи строится по одному шаблону. Это нужно, чтобы пользователь читал любую страницу одинаково и быстро находил нужное.

```mdx
---
title: "Название раздела"
description: "Один абзац: что это и для кого"
sidebarTitle: "Короткое название"
keywords: ["ключевые", "слова"]
---

Один-два абзаца: что это, зачем нужно, кто это использует.

<Frame>
  Сюда скриншот: главный экран раздела.
</Frame>

## Что вы можете сделать

<CardGroup cols={2}>
  <Card title="Действие 1" icon="..." />
  <Card title="Действие 2" icon="..." />
</CardGroup>

## Как это сделать

<Frame>
  Сюда видео: 60-секундная демонстрация раздела.
</Frame>

<Steps>
  <Step title="Шаг 1">Описание действия.</Step>
  <Step title="Шаг 2">Описание действия.</Step>
  <Step title="Шаг 3">Описание действия.</Step>
</Steps>

## Частые ошибки

<AccordionGroup>
  <Accordion title="Что-то не работает">Решение.</Accordion>
</AccordionGroup>

## Что дальше

<CardGroup cols={2}>
  <Card title="Связанная тема" icon="..." href="/ru/..." />
</CardGroup>
```

### Правила использования плейсхолдеров

- Каждый раздел должен содержать **минимум один** `<Frame>`-плейсхолдер для скриншота — даже если скриншота ещё нет, мы помечаем место.
- Если действие выполняется по шагам, добавляем **отдельный** `<Frame>` для видео-инструкции (60-90 секунд).
- Подпись внутри плейсхолдера всегда начинается со слов `Сюда скриншот:` или `Сюда видео:`, чтобы при поиске по репозиторию легко находить незаполненные места.
- Видео заливаем в `/images/videos/` и встраиваем как `<video>` или через Frame с iframe.

### Чек-лист готовности страницы

- [ ] Заголовок `title` не дублируется с другой страницей.
- [ ] `sidebarTitle` короткий (1-3 слова) и уникален в своей группе.
- [ ] Лид-абзац отвечает на «что это и зачем».
- [ ] Есть хотя бы один `<Frame>` под скриншот.
- [ ] Есть `<Frame>` под видео, если на странице описаны действия.
- [ ] В конце есть блок «Что дальше» со ссылками.
- [ ] Никакого AI-жаргона: «production-сценарий», «офисный сценарий», «клиентский сценарий», «embed-код» — заменить на человеческие формулировки.
- [ ] Ссылки на кабинет ведут на `https://app.gridix.live`, не на `gridix.app`.

## Терминология

| Использовать | НЕ использовать | Почему |
| --- | --- | --- |
| Агентская сеть девелопера | Брокерская сеть | Это направление продукта, не профессия |
| Партнёрская программа GRIDIX | Партнёр застройщика для рекомендации | Это отдельный продукт — рекомендация платформы |
| Агентство, брокер, агент | Партнёр застройщика | Это конкретные роли внутри агентской сети |
| Кабинет агентства / брокера | Кабинет партнёра застройщика | Чтобы не путать с партнёрской программой |
| Центр помощи GRIDIX | База знаний GRIDIX | Стандарт для help-центров уровня Notion, Stripe, Linear |
| Войти в кабинет → app.gridix.live | gridix.app, sso.gridix.live | Реальный домен авторизации |
| Сайт платформы → gridix.live | gridix.app | Маркетинговый сайт |
